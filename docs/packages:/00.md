**Package 00: Dev Environment + Project Scaffold**

**Goal:** Running Postgres, FastAPI health endpoint, Alembic initialized, pytest working.

**Tooling decisions (can't be inferred):**
- Python 3.13, Node 24
- `uv` for Python packages (not pip/poetry)
- Postgres 17 in Docker via docker-compose; FastAPI and Vite run natively
- Alembic for migrations (will autogenerate from SQLAlchemy models later)
- Factory Boy + pytest for tests
- pydantic-settings for config (`Settings` class reads from `.env`)

**File structure:**
```
portfolio/
├── backend/
│   ├── app/
│   │   ├── __init__.py
│   │   ├── main.py          # FastAPI app, health endpoint
│   │   ├── config.py        # Settings class
│   │   └── db.py            # Engine, session factory
│   ├── tests/

│   ├── alembic/
```

**Acceptance criteria:**
1. `docker compose up -d` starts Postgres
2. `uv run uvicorn app.main:app --reload` returns 200 on `GET /health`
3. `uv run alembic upgrade head` runs (empty migration OK)
4. `uv run pytest` passes (one trivial test)

**Gotchas:**
- DATABASE_URL needs `+asyncpg` for async SQLAlchemy: `postgresql+asyncpg://...`
- Don't add models yet — Package 01 does that with RLS
- `.env.example` committed, `.env` gitignored
- DB user for app must NOT be superuser — superusers bypass RLS
- Create dedicated app user: GRANT SELECT, INSERT, UPDATE, DELETE on tables, no BYPASSRLS
- All primary keys: UUID (use uuid7 for sortability)
- All tables: created_at TIMESTAMPTZ DEFAULT now(). Tables with user mutations add updated_at, updated_by.
- All monetary/quantity columns: NUMERIC(19,4). Serialize as string in JSON.
